## Зачем принимать крипту

Короткий ответ: деньги, которые вы не принимаете — это деньги, которые вы теряете.

```
Аудитория крипто-пользователей (глобально):

2020  ████████░░░░░░░░░░░░░░  ~100 млн
2022  ████████████░░░░░░░░░░  ~320 млн
2024  ██████████████████░░░░  ~560 млн
2026  ██████████████████████  ~700+ млн (оценка)
```

Но дело не только в размере аудитории. Вот конкретные причины:

```
Чарджбэки             Крипто-платёж необратим. Нет чарджбэков.
                      Для продавцов цифровых товаров это спасение.

Комиссии              Банковский эквайринг: 2.5–3.5%
                      Крипто-процессинг: 0.3–1%
                      Прямой кошелёк: 0% (только сетевая комиссия)

География             Карта может не работать из другой страны.
                      Крипта работает отовсюду.

Скорость              Банковский перевод: часы–дни
                      USDT TRC-20: 5–30 секунд
                      Bitcoin Lightning: мгновенно

Анонимность           Часть аудитории принципиально не хочет
                      привязывать карту к покупкам
```

Отдельный момент — аудитория с крипто-кошельками в среднем более платёжеспособна. По статистике площадок, средний чек крипто-покупателя на 30–40% выше, чем у покупателя с карты.

## Какие монеты принимать

Не нужно принимать всё подряд. Вот что реально используют для платежей в 2026 году:

### Tier 1 — обязательно

| Монета | Сеть | Комиссия сети | Скорость | Зачем |
|---|---|---|---|---|
| **USDT** | TRC-20 | ~0.5–1$ | 5–30 сек | Стейблкоин №1, нет волатильности |
| **USDT** | ERC-20 | 2–10$ | 1–5 мин | Для тех кто на Ethereum |
| **BTC** | Lightning | <0.01$ | Мгновенно | Самая известная крипта |

### Tier 2 — желательно

| Монета | Сеть | Комиссия сети | Скорость | Зачем |
|---|---|---|---|---|
| **USDC** | Различные | 0.5–5$ | 30 сек–5 мин | Второй стейблкоин |
| **TON** | TON | ~0.01$ | 5 сек | Популярен в СНГ из-за Telegram |
| **LTC** | Litecoin | ~0.01$ | 2–5 мин | Быстрый, дешёвый |

### Tier 3 — опционально

| Монета | Зачем |
|---|---|
| **ETH** | Для Ethereum-пользователей |
| **XMR (Monero)** | Для аудитории, которой важна полная приватность |
| **SOL** | Растущая экосистема |

### Практическая рекомендация

```
Минимум:     USDT TRC-20 + BTC
Оптимум:     USDT TRC-20 + USDT ERC-20 + BTC Lightning + TON
Максимум:    Всё из Tier 1 + Tier 2

Если принимаете одну монету — пусть это будет USDT TRC-20.
90% крипто-платежей в рунете проходят через неё.
```

## Способы приёма платежей

### 1. Прямой кошелёк

```
Вы:          Генерируете адрес кошелька
Покупатель:  Отправляет крипту на этот адрес
Вы:          Проверяете поступление в блокчейне
Система:     Выдаёт товар

Плюсы:
+ Комиссия 0% (только сетевая)
+ Полный контроль над средствами
+ Никаких посредников

Минусы:
- Нужен мониторинг блокчейна
- Нужна генерация уникальных адресов для каждого заказа
- Нет автоматической конвертации в фиат
- Нужно самому решать вопрос с волатильностью
```

### 2. Процессинговый сервис

```
Вы:          Подключаете API процессинга
Покупатель:  Попадает на страницу оплаты процессинга
Процессинг:  Принимает крипту, проверяет, уведомляет вас
Система:     Выдаёт товар

Плюсы:
+ Всё автоматизировано
+ Можно получать выплаты в фиате
+ Поддержка множества монет из коробки
+ Автоматическая конвертация

Минусы:
- Комиссия 0.3–1%
- Зависимость от сервиса
- KYC/AML требования
```

### 3. Через маркетплейс

```
Покупатель оплачивает на площадке → площадка обрабатывает → вы получаете выплату

Плюсы:
+ Ноль интеграции
+ Площадка берёт на себя процессинг, фрод и споры
+ Часто есть гарант-сервис

Минусы:
- Комиссия площадки (обычно выше чем у процессинга)
- Привязка к конкретной платформе
```

## Волатильность и как с ней жить

Главный страх продавцов: «а вдруг BTC упадёт на 20% пока я вывожу?»

### Стратегии

```
Стратегия 1: Мгновенная конвертация
─────────────────────────────────────
Получил крипту → процессинг автоматически конвертирует в USDT/фиат
Волатильность: 0%
Минус: комиссия конвертации (~0.3–0.5%)


Стратегия 2: Приём в стейблкоинах
─────────────────────────────────────
Принимаете только USDT/USDC
Волатильность: 0% (привязка к доллару)
Минус: отсекаете аудиторию без стейблкоинов


Стратегия 3: Hold
─────────────────────────────────────
Получил BTC → держишь
Волатильность: максимальная
Подходит: если верите в рост


Стратегия 4: Гибрид
─────────────────────────────────────
50% мгновенно конвертируете в фиат
50% оставляете в крипте
Баланс между стабильностью и потенциальным ростом
```

> **Рекомендация:** если вы не криптоинвестор — принимайте в USDT или включайте мгновенную конвертацию. Вы продавец товаров, а не трейдер. Не добавляйте себе ещё один источник стресса.

## Налоги и легальность

Важно: это не юридическая консультация. Обратитесь к своему бухгалтеру.

### Общая картина (РФ, 2026)

```
Статус крипты:           Цифровой актив (не валюта)
Приём как оплата:        Формально — серая зона
Налогообложение:         НДФЛ 13–15% с дохода от продажи крипты
Отчётность:              Обязательна при превышении порогов
Штрафы за неотчётность:  Существуют

Практика:
Большинство мелких продавцов   →  не декларируют (на свой риск)
Средний бизнес                 →  используют ИП/ООО + процессинг с KYC
Крупный бизнес                 →  полная отчётность через процессинг
```

### Как легализовать

```
Вариант 1: Процессинг с фиат-выплатами
Процессинг принимает крипту → конвертирует → выводит фиат на р/с ИП
Вы работаете только с фиатом. Крипту не касаетесь.

Вариант 2: ИП на УСН 6%
Доход от крипто-продаж декларируете как доход от реализации.
Платите 6% от выручки. Просто и понятно.

Вариант 3: Самозанятый
До 2.4 млн ₽/год. Платите 4–6% от дохода.
Подходит для начала.
```

## Интеграция в магазин

### Webhook-схема (универсальная)

```
1. Покупатель выбирает «Оплатить криптой»
2. Ваш backend создаёт инвойс через API процессинга
3. Процессинг возвращает адрес + сумму + QR-код
4. Покупатель отправляет крипту
5. Процессинг отслеживает транзакцию
6. При подтверждении → webhook на ваш сервер
7. Ваш сервер проверяет подпись → выдаёт товар
```

### Пример создания инвойса (CryptoCloud API)

```python
import requests

def create_invoice(order_id, amount_usd):
    response = requests.post(
        'https://api.cryptocloud.plus/v2/invoice/create',
        headers={
            'Authorization': f'Token {API_KEY}',
            'Content-Type': 'application/json'
        },
        json={
            'amount': amount_usd,
            'currency': 'USD',
            'order_id': str(order_id),
            'email': 'optional@email.com'
        }
    )
    
    data = response.json()
    return {
        'payment_url': data['result']['link'],
        'invoice_id': data['result']['uuid']
    }
```

### Пример обработки webhook

```python
from fastapi import FastAPI, Request
import hmac, hashlib

app = FastAPI()

@app.post('/webhook/cryptocloud')
async def handle_webhook(request: Request):
    body = await request.body()
    signature = request.headers.get('X-Signature')
    
    # Проверка подписи
    expected = hmac.new(
        SECRET_KEY.encode(), body, hashlib.sha256
    ).hexdigest()
    
    if signature != expected:
        return {'error': 'invalid signature'}, 403
    
    data = await request.json()
    order_id = data['order_id']
    status = data['status']
    
    if status == 'success':
        await deliver_product(order_id)
        await notify_buyer(order_id)
    
    return {'ok': True}
```

## Фрод в крипте

Крипто-платежи необратимы — это защищает от чарджбэков, но создаёт другие проблемы.

### Типичные схемы

```
1. Даблспенд (только BTC on-chain без подтверждений)
   Мошенник отправляет BTC → вы видите транзакцию → 
   выдаёте товар → мошенник отменяет транзакцию через RBF
   
   Защита: ждать минимум 1 подтверждение (BTC),
   или использовать Lightning / стейблкоины

2. Пылевые атаки
   Отправка микротранзакций на ваш адрес для 
   деанонимизации и анализа кошелька
   
   Защита: новый адрес на каждый заказ, 
   не консолидировать мелкие UTXO

3. Фейковые токены
   На кошелёк приходит «USDT» — но это фейковый токен
   с таким же названием на другом контракте
   
   Защита: проверять адрес контракта, 
   не доверять названию токена в кошельке

4. Неполная оплата
   Покупатель отправляет $9.50 вместо $10.00
   и требует товар «ну я же заплатил почти всё»
   
   Защита: автоматическая проверка точной суммы,
   отклонение при недоплате > порога (обычно 1–2%)
```

## Конверсия: крипто vs фиат

Что происходит с конверсией при добавлении крипто-оплаты:

```
Тип магазина                До крипты    После крипты    Изменение
──────────────────────────────────────────────────────────────────
Цифровые товары (глобал)    2.1%         2.8%            +33%
VPN/прокси                  3.4%         5.1%            +50%
Игровые товары              1.8%         2.4%            +33%
Услуги (фриланс)            1.5%         1.9%            +27%
Физические товары            2.0%         2.2%            +10%

Источник: усреднённые данные из открытых кейсов
```

> Наибольший прирост — в нишах, где аудитория технически грамотная и ценит приватность. Для физических товаров массового спроса эффект минимальный.

## Заключение

Крипто-платежи в 2026 году — это не эксперимент, а рабочий инструмент. Особенно для цифровых товаров, где основные боли — чарджбэки, география и скорость.

Что делать:

```
1. Начните с USDT TRC-20 — покрывает 90% крипто-аудитории
2. Используйте процессинг если не хотите разбираться в блокчейне
3. Включите мгновенную конвертацию если пугает волатильность
4. Не забывайте про налоги — серая зона сужается каждый год
5. Не принимайте BTC on-chain без подтверждений
6. Каждый платёж — отдельный адрес
7. Проверяйте контракт токена, а не его название
```

Необратимость крипто-платежей — это суперсила для продавца и риск для покупателя. Поэтому если продаёте через крипту — дайте покупателю гарантию. Гарант-сервис, прозрачные правила, быстрый арбитраж. Это не благотворительность — это инвестиция в повторные продажи.
